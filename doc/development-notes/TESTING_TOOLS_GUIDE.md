# EPub-InkPlate Testing Tools Guide

This document lists the testing tools currently available in this repository and explains how to use each one.

## 1) Linux Unit Test Runner (`epub_test`)

Primary unit/integration test binary for Linux-hosted development.

Source of suite registration: `test/test_runner.cpp`.

### Build

```bash
cd /home/turgu1/Dev/EPub-InkPlate
make build_test
```

### Run all suites

```bash
make test
# or
./build_test/epub_test
```

### Run one suite

```bash
./build_test/epub_test unzip
./build_test/epub_test epub
```

### Run multiple suites

```bash
./build_test/epub_test himem dom css
```

### Available suites

- `himem`
- `himem_pool_test`
- `dom`
- `simple_db`
- `css`
- `display_list`
- `app_config`
- `epub`
- `unzip`
- `simple_list`
- `char_pool`
- `fonts_cache`
- `fonts_cache_stress`
- `gif_decoder`
- `svg_decoder`

### Useful make shortcuts

```bash
make test_unzip
make test_epub
make test_svg_decoder
make test_gif_decoder
make clean_test
```

### Notes

- Unknown suite names exit with code `2` and print available suite names.
- The test binary uses a mix of real and stubbed components (for fast host-side testing).

## 3) Valgrind on Linux GUI Binary (`valgrind_linux`)

Runs the Linux GUI executable under Valgrind with GTK suppressions.

### Build debug binary first

```bash
cd /home/turgu1/Dev/EPub-InkPlate
make DEBUG=1
```

### Run

```bash
make valgrind_linux
```

### Optional book override

```bash
make valgrind_linux VALGRIND_EPUB="/absolute/path/to/book.epub"
```

### Output

- Valgrind output is printed to terminal.
- Suppressions file: `gtk3.supp`.

## 4) Headless S5 Valgrind Harness (`epub_valgrind_s5`)

Dedicated memory/leak/concurrency harness for Scenario 5 (concurrent navigation + pageLocs restart), including current preflight checks.

Source: `test/linux_s5_valgrind.cpp`.

### Build

```bash
cd /home/turgu1/Dev/EPub-InkPlate
make valgrind_s5
```

### Run directly (no Valgrind)

```bash
./build_valgrind/epub_valgrind_s5 test/fixtures/minimal.epub
```

### Run with timeout safety

```bash
timeout 180 ./build_valgrind/epub_valgrind_s5 test/fixtures/minimal.epub
```

### Run with Valgrind instrumentation

```bash
make valgrind_run
```

### Override input EPUB for `valgrind_run`

```bash
make valgrind_run VALGRIND_EPUB="/absolute/path/to/book.epub"
```

### Clean

```bash
make clean_valgrind
```

### Output

- Runtime logs printed to stderr/stdout.
- Existing sample report file in repo root: `report.valgrind`.

## 5) PageLocs Hardening Stress Test (Standalone)

Stress-testing framework for PageLocs protocol and synchronization behavior.

Source: `test/test_pagelocs_hardening.cpp`.

### Build

```bash
cd /home/turgu1/Dev/EPub-InkPlate
g++ -std=c++17 -pthread test/test_pagelocs_hardening.cpp -o build/test_pagelocs_hardening
```

### Run

```bash
./build/test_pagelocs_hardening
```

### What it checks

- Concurrent navigation behavior
- Rapid stop/start cycles
- Queue saturation behavior
- Map lock contention metrics
- Protocol deviation counters (mismatch/failure telemetry)

## 9) Recommended Testing Workflow

For normal feature work:

1. Fast validation:

```bash
make build_test
./build_test/epub_test <relevant_suite_names>
```

2. Full host suite:

```bash
make test
```

3. Memory/concurrency validation for pageLocs-heavy changes:

```bash
make valgrind_s5
./build_valgrind/epub_valgrind_s5 test/fixtures/minimal.epub
make valgrind_run VALGRIND_EPUB="/absolute/path/to/real_book.epub"
```

4. Optional deep stress:

```bash
g++ -std=c++17 -pthread test/test_pagelocs_hardening.cpp -o build/test_pagelocs_hardening
./build/test_pagelocs_hardening
```

## 10) Troubleshooting

- `Unknown suite: "..."` from `epub_test`:
  - Use one of the registered suite names listed above.
- Build failures in valgrind targets:
  - Rebuild target from scratch: `make clean_valgrind && make valgrind_s5`.
- Missing fixtures:
  - Ensure test files under `test/fixtures/` are present and command is run from repository root.
- Long-running harnesses:
  - Use `timeout` to avoid hanging CI jobs.
